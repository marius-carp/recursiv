## Project Overview

Recursiv is a serverless, zero-friction math worksheet generator for elementary school teachers. It's a single-page React application that generates printable math worksheets with automatic answer keys.

## Development Commands

```bash
# Start dev server (http://localhost:5173)
npm run dev

# Build for production
npm run build

# Lint code
npm run lint

# Preview production build
npm run preview
```

## Tech Stack

- **React 19** with JSX
- **Vite 8** for build tooling
- **Tailwind CSS 3** for styling with custom Fluent-inspired design tokens
- **oxlint** for linting (configured for React rules)
- **No test framework** currently configured

## Architecture

### Component Structure

The app uses a two-panel layout:

1. **Sidebar** (320px fixed): Configuration controls for worksheet generation
2. **WorksheetPreview** (flexible): Live preview and print-ready output

State is managed entirely in `App.jsx` and passed down to both components. There is no state management library, routing, or backend.

### Question Generation Logic

All math generation happens client-side in `App.jsx`:

- **Addition/Multiplication**: Simple random number generation within range
- **Subtraction**: `num2` is constrained to `≤ num1` to avoid negative results
- **Division**: Generated by `divisor × quotient = dividend` to ensure whole number answers (no remainders)

Questions are randomly selected from enabled operations based on `totalQuestions` count.

### Print Architecture

The app uses CSS print media queries with a dual-render approach:

- **Screen**: Shows preview with `.print-hidden` class
- **Print**: Renders both student worksheet and answer key with `.print-page-break` between them
- **Layout modes** affect both screen and print:
  - `full`: Single full-page worksheet
  - `half`: Two half-page sections with dashed borders
  - `quarter`: Four quarter-page sections with dashed borders

For half/quarter layouts, only one master answer key prints (not one per section).

## Design System

Custom Tailwind theme extends with:

- **Colors**: Microsoft Fluent-inspired palette (defined in `tailwind.config.js`)
- **Typography**: 
  - `font-inter` for UI controls
  - `font-serif` (Source Serif 4) for worksheet content
- **Spacing**: `sidebar` utility = 320px
- **Print styles**: Defined in `@layer utilities` in `index.css`

## Important Patterns

### Layout Mode Behavior

The `layoutMode` prop controls:
1. Number of questions per column (`questionsPerColumn`)
2. Grid layout structure
3. Border styling (dashed borders for half/quarter modes)
4. Font sizes (smaller for half/quarter modes)

### Print Margins

Worksheets are sized for US Letter (8.5" × 11") with dynamic padding:
- **Full layout**: 0.5in top, 1in right/bottom/left
- **Half layout**: 0.5in top, 1in right/bottom/left  
- **Quarter layout**: 1in on all sides

The reduced top padding for full/half layouts ensures all questions fit within a single printed page.

### State Dependencies

When changing `operations`, `range`, or `totalQuestions`, call `generateQuestions()` explicitly. The initial generation happens in `useEffect` on mount.
